Accessibility of What’s on GOV.UK

This is an independent, derived metadata catalogue. GOV.UK remains the authoritative destination.

Current test status

The development fixture has an automated real-browser harness for landmarks and accessible names, keyboard focus, narrow reflow, reduced motion, forced colours, contrast-relevant computed styles, hash deep links, gzip data loading and performance budgets. That historical regression evidence is not evidence about a later release corpus.

A full-corpus site is published only after separate snapshot-bound browser checks cover the same automated subset, routing, exact packed data loading and performance budgets. When this page belongs to a verified release, its exact release evidence link is resolved from the signed-off local data-plane manifest.

This is not a claim of WCAG 2.2 AA conformance. Automated testing does not replace accessibility-expert review, screen-reader testing or authorised representative-user research.

How the Explorer is designed to be accessible

  • semantic header, navigation, search, main and footer landmarks;
  • native links, buttons, form controls, tables and disclosure controls;
  • a keyboard skip link and visible focus treatment;
  • layouts that reflow at a 320 CSS-pixel viewport, representing 400% zoom on a 1280-pixel display;
  • reduced-motion and forced-colour adaptations;
  • a non-visual list or table for every relationship shown in the graph;
  • source, lifecycle and uncertainty information that does not depend on colour alone.

Known limitations

  • Automated browser evidence covers a bounded technical subset; it cannot establish full assistive-technology or cognitive accessibility.
  • No authorised participant study has been run, so the project does not claim that this is a preferred human interface.
  • Automated checks cannot establish screen-reader usability, cognitive accessibility or full WCAG conformance.
